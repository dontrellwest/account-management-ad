# Active Directory: Account Management and Log Observation

I used the existing Active Directory environment to work through common account management scenarios that come up regularly in help desk work: account lockouts, password resets, disabling and re-enabling accounts, and reading event logs.

---

## Environment

- Microsoft Azure (DC-1 and Client-1 from the Active Directory lab)
- Windows Server 2022 (DC-1)
- Windows 10 (Client-1)
- Active Directory Users and Computers
- Group Policy Management
- Windows Event Viewer

---

## What I Did

### Account Lockout

Picked a user account and attempted to log in with the wrong password 10 times. Configured a Group Policy to lock accounts after 5 failed attempts, then repeated the test. The account locked as expected after the fifth bad attempt.

Located the locked account in Active Directory, unlocked it, reset the password, and confirmed the user could log back in.

### Disabling and Re-enabling Accounts

Disabled the same account in Active Directory and attempted to log in. The error message was different from a lockout, which matters when triaging access issues over the phone. Re-enabled the account and confirmed access was restored.

### Observing Logs

Pulled up Event Viewer on both DC-1 and Client-1 and looked at the security logs generated by the failed login attempts, the lockout, and the account changes. Each action left a traceable record.

---

## Key Takeaways

Lockouts and disabled accounts produce different error messages on the login screen. Knowing the difference saves time when a user calls in saying they can't log in. The log observation piece ties into security operations: every one of those failed login attempts is recorded, and knowing where to look for that trail is a foundational skill.
